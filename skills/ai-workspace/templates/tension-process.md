# Tension Process

Use the user's workspace language for headings, filenames, and path segments.
`tension` is the semantic layer name, not a required English filename.

## Purpose

Identify unresolved pressure, contradiction, risk, or opportunity.

Tension answers:

- what is misaligned?
- what needs attention?

## Inputs

Allowed inputs:

- source evidence;
- current or new memory layer file;
- existing current tension layer file;
- dated tension snapshots.

## Rules

- Name the pressure clearly.
- Distinguish risk from opportunity.
- Prefer tensions that imply a decision.
- Do not inflate weak signals.
- State the domain mechanism.

## Output

```md
## Tensions

- Tension: ...
- Type: risk | opportunity | contradiction | pressure

## Supporting Evidence

- ...

## Severity

High | Medium | Low

## Why It Matters

- ...

## Domain Mechanism

- ...
```

## TTL

Default TTL: `P7D`.
